# Convention Categories

Inspect these categories in both source and target repos.

## Repo Instructions And Process

- `AGENTS.md`, `CLAUDE.md`, `.agents/`, `.claude/`
- plan directories and planning templates
- contribution docs and engineering process notes

## Toolchain And Environment

- version managers like `mise.toml`
- language toolchain pins
- dependency manager config
- local dev shell or setup scripts

## Build, Test, Lint, Format

- build entrypoints
- formatter config
- linter config
- test commands and test layout
- coverage or static analysis
- pre-commit, lefthook, or other local hooks

## CI And Automation

- GitHub Actions workflows
- reusable workflow fragments
- CI helper scripts
- validation jobs, matrix builds, cross-compilation

## Release And Distribution

- release workflows
- changelog generation
- packaging scripts
- registries and publishing config
- version-tagging conventions

## Collaboration Conventions

- issue templates
- pull request templates
- CODEOWNERS
- labels, triage docs, bug-reporting patterns

## Docs And Knowledge Layout

- docs structure
- README patterns
- examples and sample configs
- architecture docs

## Repo Hygiene

- `.gitignore`
- `.editorconfig`
- formatter ignore files
- generated-file conventions
- schema validation hooks or config validation

## Inventory Output

Produce at least:
- source-only conventions
- target-only conventions
- overlapping conventions
- obvious conflicts
- conventions that look source-product-specific
